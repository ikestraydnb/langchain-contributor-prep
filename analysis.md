# Issue #33504 Analysis

**Title:** create_agent Does Not Handle invalid_tool_calls from JSON Parsing Errors (langchain alpha v1)

**Description Summary:**
- The `create_agent` function does not handle `invalid_tool_calls` generated by JSON parsing errors. When the LLM returns malformed JSON, `last_ai_message.tool_calls` is empty, causing the agent to exit the loop without feeding the error back to the model.
- The routing logic in `langchain/agents/factory.py` only checks `tool_calls` and ignores `invalid_tool_calls`.
- The `ToolNode` also ignores `invalid_tool_calls`.
- Previous `AgentExecutor` had a `handle_parsing_errors` flag to convert these errors into `ToolMessage`s, but this was not ported to `create_agent`.
- Current workaround uses custom middleware to convert `invalid_tool_calls` to `ToolMessage`s, but routing still exits early.
- Suggested fix: Update routing to consider `invalid_tool_calls` or add a built‑in middleware/parameter to `create_agent` for automatic handling.

**Impact:** High – agents silently fail on parsing errors, leading to poor developer experience.

**Potential Contribution:**
- Implement handling of `invalid_tool_calls` in the routing logic.
- Add an optional `handle_parsing_errors` parameter to `create_agent`.
- Write tests for the new behavior.
